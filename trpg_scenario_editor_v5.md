# TRPGシナリオエディタ 概念設計 v5

v4 にプロトタイプ検証（v1〜v8b、8回のイテレーション）の知見を統合した設計文書。

---

## 設計思想

- **常に編集可能。** セッション中にシナリオ構造を自由に変更できる。即興を阻害しない。
- **エンティティの統一。** 場所・人物・アイテムを区別しない。すべてエンティティ。
- **トリガーは支援であり制約ではない。** 事前設計されたトリガーを消化しなくても進行できる。
- **KPの操作速度が最優先。** セッション中のKPの認知負荷と操作コストを最小化する。
- **KPが忘れることを前提にする。** 評価すべきタイミングをシステムが通知する。
- **場面中心。** KPの視点は「場所で何が起きているか」であり、個別エンティティの状態ではない。
- **描写が成果物。** 状態管理はKPのためのもの。PLに届く成果物は描写テキストであり、セッションの記録でもある。

## データモデル

### エンティティ

シナリオを構成するすべての要素。ツリー構造（親子関係）を持つ。

```
エンティティ {
  名前
  親: エンティティへの参照（nullable）
  カテゴリ集合
  アクション集合
  トリガー集合
  描写カテゴリ順
}
```

- 場所もNPCもアイテムもPCもエンティティ。区別は種別カテゴリの値で表現される。
- 種別の値候補：場所、NPC、PC、アイテム。これはテンプレートによって初期設定される。

### ツリー

場所の包含関係を表現する。

- 場所の階層（世界→国→都市→建物→部屋→家具→中身）を自然なネストで表現する。深いネスト（10段以上）を許容する。
- NPCやアイテムは現在いる場所の下に配置される。
- エンティティの分類・整理にはカテゴリを使う。ツリー上にグルーピング用フォルダは作らない。
- 相対参照（祖先・子孫・同位）の基盤。
- 左ペインのナビゲーション構造。PCの現在地を●で表示する。

### 居場所カテゴリ

すべてのエンティティは暗黙の排他カテゴリ「居場所」を持つ。この値は常にツリー上の親エンティティと同期する。

- KPが居場所カテゴリの値を変更すると、ツリー上の親が変わる。
- ツリー上でドラッグ移動すると、居場所カテゴリの値が変わる。
- アクション結果に `居場所 → 応接間` と書くことで移動を表現できる。新しい操作型は不要。
- トリガー条件で `[居場所:書斎]` のように参照できる。

### カテゴリ

エンティティの状態の「軸」。値定義と現在値を持つ。

```
カテゴリ {
  名前: "態度"
  排他: true
  値定義: {
    "中立":    { 描写: null },
    "表面的警戒": { 描写: "こちらを値踏みするような目で見ている" },
    "明確な敵意": { 描写: "明らかな敵意を隠そうともしない" },
    "協力的":   { 描写: "穏やかに微笑んでいる" }
  }
  現在値: "中立"
}
```

- 排他カテゴリは同時に1つの値。非排他カテゴリは値の集合（タグ）。
- 値定義はドロップダウンUIに使われる。自由入力で追加でき、追加された値は自動的に候補に加わる。
- 排他カテゴリへの値の付与は旧値を自動的に除去する。
- 描写を持つ値はPL向け表示に寄与する。描写を持たない値はPL向け表示に出ない。

#### 知識カテゴリ

PCエンティティが持つ非排他カテゴリ。PCが知っている情報をタグとして蓄積する。

- アクションの結果で行為者に知識が付与される（例：日記を読む → 行為者に「日記の内容」知識が付与される）。
- 知識を持つPCだけが特定のアクションの行為者候補になる（requiresKnowledge）。
- 同じ場所にいるPC間で情報共有できる。共有はコピーであり、共有元は知識を失わない。

### アクション

物語上の出来事。KPがボタン一つで発火する、またはトリガーによって自動発火する。

```
アクション {
  名前: "本棚を調べる"
  描写: "本棚の奥に、何か挟まっているのが見える。古い革張りの日記が出てきた。"
  結果: [付与/除去 操作の列]
  PLアクション: true
  ロール条件: "目星 50"
  表示条件: [カテゴリ:値 の条件列]
  必要アイテム: エンティティへの参照（nullable）
  必要知識: 値（nullable）
}
```

#### 操作

```
操作 = 付与 { 対象エンティティ, カテゴリ, 値 }
     | 除去 { 対象エンティティ, カテゴリ, 値 }
```

- 対象エンティティを省略した場合、アクションが属するエンティティを対象とする。
- 対象エンティティに `$actor` と書くと、実行時に行為者のエンティティIDに置換される。値にも `$actor` を使用可能（例：アイテムの居場所を行為者に移す）。
- 任意のエンティティの値を変更できる。書き込みスコープの制約はない。
- 居場所カテゴリの付与操作はツリー上の移動として反映される。

#### 描写

アクション発火時にKPに表示される動的な描写テキスト。PLに読み上げるための文章。

カテゴリの値に紐づく描写（静的描写）は「今どう見えるか」を表し、アクションの描写（動的描写）は「今何が起きたか」を表す。両方が必要。

#### PLアクションとKP判断

PLアクション（plAction: true）はPLの行動結果として提示されるアクション。行為者（どのPCがやるか）の選択が発生する。

KP判断（plAction: false）はKPの裁量で発火するアクション。NPC行動やイベントなど。行為者の選択は不要。

UIではPL行動とKP判断が別のグループで表示される。

#### 表示条件

条件が満たされないアクションはUI上に表示されない。条件式はトリガーの条件と同じ形式。

用途：「日記を読む」は日記が発見済みのときだけ表示。「執事が逃走する」は執事が動揺しているときだけ表示。

#### ロール条件

テキストでダイス判定を提示する。判定自体はKPが行い、成功ならアクションボタンを押す。システムは判定を強制しない。

#### 必要アイテム・必要知識

PLアクションの行為者候補を絞り込む。必要アイテムを所持している（ツリー上でそのPCの子である）PC、または必要知識を持つPCだけが行為者候補になる。

### トリガー

アクションに条件を紐づけ、条件が揃えば自動でアクションを実行するルール。トリガーは実行するアクションが属するエンティティに属する。

```
トリガー {
  条件: [カテゴリ:値 の条件列]
  実行: アクションへの参照
}
```

- すべて自動発火。手動トリガーという概念はない。KPの手動判断はアクションの直接発火に集約される。
- ORが必要な場合は同一アクションを参照する複数のトリガーで表現する。

### パーティ

PCおよび同行NPCのグループ。移動と行為者選択の単位。

```
パーティ {
  名前: "パーティ" or "別動隊"
  メンバー: [エンティティIDの列]
}
```

- 通常はパーティは1つ。全PCと同行NPCが含まれる。
- パーティ移動：場所ナビゲーションでの移動はアクティブパーティ全員が移動する。
- 分割：KPがメンバーを選択して別パーティに分ける。
- 合流：同じ場所にいるパーティ同士を統合する。
- パーティ切替：アクティブパーティを切り替えると、そのパーティの現在地にビューが切り替わる。
- NPCもパーティに追加可能。場所の「状態」セクションから「+パーティ」ボタンで追加。

## トリガーの条件：相対参照と名前指定参照

### 相対参照（4種類）

```
自身    → 自エンティティ
祖先    → 自身を含む場所ツリーを根に向かって探索
子孫    → 自身の下位にあるすべてのエンティティ
同位    → 自身と同じ親を持つエンティティ（自身を除く）
```

### 名前指定参照

特定のエンティティをIDで指定する。UI上は名前で表示。名前変更時は参照が自動追従。

## 状態変更とトリガー評価

### アクション発火時の処理（stabilize）

```
1. アクションの結果（値変更）を適用する（$actorを行為者IDに置換）
2. 変更された値を条件に含むトリガーをシナリオ全体から検索する
3. 該当トリガーの全条件を評価し、すべて成立し、かつ結果が状態を変える場合にアクションを発火する
4. 新たに発火したアクションについて 1〜3 を繰り返す（不動点計算）
5. 新たに発火するトリガーがなくなったら安定
```

- トリガーの検索範囲はシナリオ全体。エンティティの開閉やツリー上のスコープに依存しない。
- 同一stabilizeループ内で同じトリガーは2度発火しない。

### 場面を開いたときの処理（project）

```
1. 自身と子孫（ツリー上）の描写を持つ値を収集する
2. カテゴリの表示順序に従って描写テキストを合成する
```

場面を開く操作は描写合成のみ。トリガー評価は行わない。

### 不動点計算の停止保証

値の付与・除去は冪等。有限の状態空間。最大ステップ数 = エンティティ数 × 値種類数。上限に達したら振動として警告、打ち切り。

### 場所の進入条件

場所エンティティは進入条件を持てる。条件が満たされない場所には移動ボタンが🔒表示になり、移動できない。

```
場所エンティティ {
  進入条件: [カテゴリ:値 の条件列]
}
```

例：地下室の進入条件 = `[施錠:解錠]`。鍵を使って解錠するまで入れない。

---

## 待機中トリガーの通知

トリガーの条件が部分的に充足されたとき（残り1条件のみ未充足のとき）、KPに通知する。

- 補助パネルに表示。未充足条件の「付与 ▶」ボタンでワンクリック発火。
- 残り1条件のみに限定することでノイズを防ぐ。
- KPの判断を代替しない。いつ条件を満たすかはKPが決める。

---

## 描写ログ：セッションの記録

アクション発火時の動的描写をセッション全体で時系列に記録する。

```
ログエントリ {
  時刻
  アクション名
  描写テキスト
  行為者名（PLアクションの場合）
  場所名
  ロール条件（あれば）
  自動発火か手動か
}
```

- ミニログとしてメインパネル下部に常時表示。
- 場所移動時、移動先の場面描写もログに追加される。
- 各エントリにコピーボタン。描写テキストをクリップボードにコピーしてPLに貼り付ける。
- セッションデータの一部として保存される。

---

## 情報共有

同じ場所にいるパーティメンバー間で知識を共有できる。

- 「情報共有」セクションに、メンバー間の知識差分が表示される。
- 「共有」ボタンで知識を他のメンバーにコピーする。
- 共有はコピーであり、共有元は知識を失わない。
- 共有は描写ログに記録される。

---

## 関係性の表現

エンティティ間の動的な関係は、カテゴリ名に相手を含めることで表現する。汎用的な関係モデルやエンティティ参照型の値は導入しない。

## KPの操作設計

### 場面中心の操作

KPの操作単位は「場面（場所）」。場所を選択すると、その場所と子孫のアクション・状態がすべて一画面に表示される。

- 場所自身のアクションに加え、子エンティティ（NPC・アイテム等）のアクションも集約表示される。エンティティ名でグルーピング。
- 孫エンティティ（NPC所持のアイテム等）のアクションも含めて表示される。

### 場所間ナビゲーション

場面上部に同階層の場所がボタンとして並ぶ。クリックするとアクティブパーティ全員がその場所に移動し、ビューが切り替わる。移動先の場面描写が自動でログに追加される。

- ツリークリックは閲覧のみ（パーティは動かない）。パーティがいない場所を閲覧中は警告を表示。
- 進入条件を満たさない場所は🔒表示で移動不可。

### 速度階層

```
最速     事前設計アクションボタン押下    ワンクリック
最速     待機中トリガーの残条件付与      ワンクリック（[付与 ▶]）
最速     場所移動（パーティ移動）       ワンクリック
速い     カテゴリの値変更（既存値）     ドロップダウン選択
やや遅い  カテゴリの新しい値入力        テキスト入力数文字
少し遅い  新規カテゴリ+値の即席作成     インラインフォーム
遅い     トリガーの編集               設計作業。セッション中でも可能。
```

### トリガーは支援であり制約ではない

v4と同じ。

## エンティティテンプレート

エンティティの新規作成時に種別を選択し、初期カテゴリセットが適用される。

```
テンプレート: 場所
  初期カテゴリ: [雰囲気]

テンプレート: NPC
  初期カテゴリ: [種別:NPC, 外見, 態度]
  初期値: [態度:中立]

テンプレート: PC
  初期カテゴリ: [種別:PC, 知識, 状態異常]
  PC作成時にアクティブパーティに自動参加。

テンプレート: アイテム
  初期カテゴリ: [種別:アイテム, 状態]
  初期値: [状態:未発見]

テンプレート: 空
  初期カテゴリなし。自由に構築。
```

- 生成後はテンプレートとの紐づけを持たない。
- UIでは種別ボタンとして並び、名前入力と合わせてワンステップで作成可能。

## データ永続化

### 形式

JSON。技術スタック非依存。

### シナリオとセッションの分離

- **シナリオデータ：** エンティティ、カテゴリ（値定義・描写を含む）、アクション（描写・表示条件・ロール条件・PLアクション区分・必要アイテム・必要知識を含む）、トリガー、ツリー構造、テンプレート、デフォルト描写カテゴリ順、場所の進入条件。
- **セッションデータ：** 各カテゴリの現在値、パーティ構成、アクションツリー（実行履歴）、描写ログ、undo用の発火前状態。

```json
{
  "scenario": {
    "name": "洋館の怪",
    "defaultCategoryOrder": ["外見", "服装", "態度", "環境", "雰囲気"],
    "templates": [
      { "name": "NPC", "categories": [...], "actions": [...] }
    ],
    "entities": [
      {
        "id": "e001",
        "name": "書斎",
        "parent": "e000",
        "categoryOrder": ["雰囲気"],
        "entryConditions": [],
        "categories": [
          {
            "name": "雰囲気",
            "exclusive": true,
            "values": {
              "静寂": { "description": "しんと静まり返っている" },
              "不穏": { "description": "どこからか微かな音がする" }
            },
            "current": "静寂"
          }
        ],
        "actions": [
          {
            "id": "a001",
            "name": "物音がする",
            "description": "どこからか不気味な物音がする。",
            "plAction": false,
            "roll": null,
            "showConditions": [],
            "requiresItem": null,
            "requiresKnowledge": null,
            "effects": [
```

## v4からの変更一覧

1. **アクション描写の追加。** アクションに動的描写テキストを持たせる。発火時にKPに表示され、PLに読み上げる内容となる。
2. **アクションの表示条件。** 条件を満たさないアクションはUI上に表示されない。
3. **アクションのロール条件。** ダイス判定をテキストで提示。判定はKPが行う。
4. **PLアクション / KP判断の区分。** アクションをPL行動とKP裁量に分け、UIで別グループ表示。
5. **行為者選択と$actor参照。** PLアクションは行為者（どのPCがやるか）を選択する。effectsの対象・値に$actorを使うと行為者IDに置換される。
6. **必要アイテム・必要知識。** 行為者候補をアイテム所持や知識で絞り込む。
7. **知識カテゴリ。** PCが知っている情報を非排他カテゴリとして管理。
8. **情報共有。** 同じ場所にいるPC間で知識をコピーで渡す。
9. **パーティ概念の導入。** 移動の単位。分割・合流。NPCも参加可能。
10. **場面中心ビュー。** 場所を開くと子孫のアクション・状態が集約表示される。
11. **場所ナビゲーション＝パーティ移動。** 同階層場所ボタンでパーティ全員が移動。ツリークリックは閲覧のみ。
12. **場所の進入条件。** 条件を満たさない場所には移動不可。
13. **描写ログ。** セッション全体の描写記録。メインパネル下部に常時表示。
14. **待機中トリガーのノイズ削減。** 未充足条件が1つだけのトリガーのみ通知。
15. **テンプレートの具体化。** 種別選択ボタン（場所・NPC・PC・アイテム・空）でワンステップ作成。
16. **インライン編集。** エンティティ・カテゴリ・アクション・トリガーの編集がすべてインラインフォームで完結。
17. **状態のコンパクト表示。** アクションの下に子エンティティの状態がドロップダウンで並ぶ。編集モードを開かずに変更可能。
